The DriverPass project created a driver test preparation system for a client who needed support for online practice tests, lesson scheduling, and instructor appointment management. The system was designed as a web-based application that allows users to access driving materials, take practice tests, and book driving sessions. The system was designed to deliver students an organized driver education program that they could use to prepare for their upcoming driving tests. 

My performance in assessing the client's requirements and transforming those needs into system specifications showed my ability to deliver effective results. I organized the requirements into separate functional parts which I documented in an organized manner. The system design I created needed less complicated elements because I concentrated on making it easy for users to interact with the system.

I must improve my diagrams and system design documentation through better explanations and additional supporting details to achieve better understanding. The system diagrams demonstrate operational functions, but I needed to create better explanations regarding how different system elements interact with each other. Design organization needs to be improved because current presentation methods create difficulty in understanding while making the design appear less professional.

The development of the system started when we collected user requirements which helped us determine the key system capabilities that include lesson scheduling and practice test access and user account management. The needs assessment created system features which we transformed into system requirements. User requirements must be understood throughout system design because they determine whether the finished product delivers actual solutions and significant benefits to users.
